## Sprint 4 · Capacitor iOS/Android [3-5 días] — "feel nativo"

### Scope
Convertir Ordy Chat en **PWA + app nativa** con Capacitor. Todas las páginas deben sentirse nativas. No solo web wrap.

### Tareas grandes (cada una con su spec+audit)

**4a. Audit de UX mobile actual** [4h]
- Screenshot every route desde móvil (Playwright mobile viewport).
- Documentar: dónde falta safe-area, dónde hay hover states que rompen, dónde input zoom, dónde transitions jerkeadas.
- Producir `docs/mobile-audit-2026-04-XX.md`.

**4b. Instalar Capacitor** [1 día]
- `@capacitor/core`, `@capacitor/ios`, `@capacitor/android`, CLI.
- `capacitor.config.ts` con appId `com.ordy.chat`.
- NO usar `output: 'export'` en Next — es incompatible con server actions / API routes.
  Usar **Capacitor Server mode** con `server.url` apuntando a `https://ordychat.ordysuite.com` (patrón PWA + Capacitor shell).
- Icons: 192, 512, maskable-512 en `public/icons/`.
- `public/manifest.webmanifest` ya existe (fase 7 onboarding fast) — revisar si tiene los icons correctos.

**4c. PWA Service Worker** [1 día]
- `public/sw.js` con cache-first para assets estáticos, network-first para API, bypass para SSE.
- `RegisterSW.tsx` montado en root layout.
- Offline fallback: pantalla "sin conexión" cacheada.

**4d. Native feel global** [1-2 días]
- Safe-area insets (`env(safe-area-inset-top/bottom)`) en header + bottom nav.
- Bottom navigation estilo app (home / conversations / dashboard / admin si super).
- Transitions entre rutas con `view-transition-name` (Next 16 soporta).
- Tap highlight off en CSS (`-webkit-tap-highlight-color: transparent`).
- Deshabilitar rubber-band scroll en áreas fijas.
- Haptic feedback via `@capacitor/haptics` en confirmaciones críticas.
- Status bar styling (`StatusBar.setStyle({ style: Style.Dark })`).

**4e. Auth Bearer token** [1 día]
- Problema: cookies Auth.js no cruzan a Capacitor nativo.
- Solución: endpoint `/api/auth/token-exchange` que devuelve JWT; Capacitor lo guarda en `@capacitor/preferences`; interceptor fetch lo adjunta en header.
- Fallback: magic link universal con custom URL scheme `ordy://auth?token=...` para nativo.

**4f. Build pipelines** [0.5 día]
- `scripts/build-mobile.mjs` — `next build` + `cap sync` + `cap open ios|android`.
- CI job que build iOS (requiere macOS runner GitHub — $ pero factible).
- TestFlight + Play Console setup (manual, fuera scope código).

**4g. Push notifications** [1 día — opcional, puede posponerse]
- `@capacitor/push-notifications` + FCM/APNs setup.
- Endpoint `/api/notifications/register-device`.
- Hook en onboarding fast + ack de mensajes de tenants.

### Flow por tarea
Cada una de las 4a-4g es su propio mini-sprint con spec+audit (excepto 4a que es audit puro, y 4f que es configuración).
